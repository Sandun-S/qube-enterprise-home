# 04 - Edge Services Deep Dive

The "Edge" is the actual physical Qube hardware sitting in a remote location. Three key pieces of software run here.

## 1. `conf-agent` (Configuration Agent)
This is a small Go binary running automatically on startup on the device.
- **Goal:** Keep the Qube in sync with the Cloud.
- **How it works:** Every 30 seconds (configurable via `POLL_INTERVAL`), it makes an HTTP request to the Cloud TP-API (`http://cloud-api:8081/v1/sync/state`).
- **If things changed:** It downloads a new `docker-compose.yml`, new CSV configuration files for the gateways, and a `sensor_map.json`.
- It then executes a shell command: `docker compose up -d` or `docker stack deploy` (if using Swarm). This transparently restarts the containers with the new configuration.
- **Initial Boot:** It reads `/boot/mit.txt` which is written at the factory. This file contains a `register_key`. It automatically registers itself to the cloud so the user can just claim the device in the UI without SSH.

## 2. `enterprise-influx-to-sql`
- **Goal:** Ship sensor data to the cloud.
- The protocol gateways read data from physical wire protocols (like Modbus). They aren't smart enough to talk to the Cloud. Instead, they dump data into a local `InfluxDB` instance running on the Qube.
- This Go application queries that local `InfluxDB` every minute.
- It uses the `sensor_map.json` file (which the `conf-agent` downloaded from the cloud) to translate raw hardware sensor names like `UPS_Main.Battery_Voltage` into Cloud Database UUIDs: `uuid-sensor-001`.
- It packages all new readings into a JSON payload and POSTs them to the Cloud TP-API `http://cloud-api:8081/v1/telemetry/ingest`.

## 3. `mqtt-gateway`
- **Goal:** Read data from MQTT brokers.
- Unlike the other Go tools which are infrastructure wrappers, this is an actual data ingestor.
- It reads a `topics.csv` file generated by the cloud. This CSV tells the gateway exactly which MQTT topics to subscribe to.
- When live data hits the topic, the gateway takes the payload, formats it into Qube's standard JSON standard, and ships it locally to the core routing system, which eventually puts it in InfluxDB.
